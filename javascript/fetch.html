# 基于axios封装api调用方法

如果在代码中直接使用axios库的方法，会导致写很多重复代码，使用不方便。

为了在业务代码中更简便的调用api，并且便于统一管理，写了此方法。

该方法通过定义一个返回promise的函数，调用此异步函数即可获得接口数据。

在一处先定义api，包括接口地址及请求方式，其后在业务代码中调用，传请求参数，实现api的简单复用。

一般来说后台都有统一的接口规范，不同服务的CRUD接口基本只有一些单词的变化，使用这种统一定义的方式，前端在对新增接口定义的时候只要复制粘贴并统一修改就可以得到，十分方便且提高工作效率。

并且可以和后台协同开发，先和后台对接好返回数据的数据接口以及接口地址，前端这里先预定义mock接口，或者使用自定义假数据，等后台接口完成即可联调，不会有互相等待造成的工作进度延后。

//定义
export const menuList = fetchData(`/menu/getMenuList`, {
  method: 'get', detail: `菜单列表`
})

//调用
import { menuList } from 'api/sys'

const res = await menuList(params)

根据method统一处理请求参数，具体需要与后台协商。

此处对请求参数的处理，分为get和其他请求方式，其他请求方式参数都放在body里传参。

request() 是基于axios请求库，添加了拦截器的方法。

export const fetchData = (url, opts = {}) => {
  const { method, description, isOper } = opts

  const ret = (reqData = {}) => {
    return new Promise((resolve, reject) => {
      const config = {
        url,
        method,
        ...getRequestData({ reqData, method })
      }
      request(config).then(res => {
        if (isOper) Vue.prototype.$baseMessage('操作成功', 'success') //统一操作成功提示
        resolve(res.data)
      }).catch(() => { reject(`${description} 接口错误`) })
    })
  }
  return ret
}

function getRequestData({ reqData, method }) {
  let ret = {}

  if (method === 'get') { //get类型-query放在params中
    if (Object.keys(reqData).length) {
      for (let key in reqData) {
        if (isEmpty(reqData[key]) || isObject(reqData[key]) || isArray(reqData[key]))
          delete reqData[key]
      }
      ret = { params: reqData }
    }
  } else {  //其余，body放在data中
    ret = { data: reqData }
  }
  return ret
}
